# 🫧 말랑해도 돼 (마음 온도계)

> **KLUE-BERT/KoELECTRA 감정 분류 모델과 GPT-4o-mini 페르소나 챗봇을 결합한 정신건강 케어 플랫폼**  
> 플레이어/사용자의 발화 데이터에서 19가지의 세부 심리 감정 및 일상 상태(총 20개 클래스)를 탐지하고, 분석된 위험 신호를 기반으로 따뜻한 맞춤형 위기 극대화 챗봇 상담 및 공공 서비스 연계를 지원하는 감정 치유 서비스입니다.

---

## 📌 프로젝트 소개 (Overview)

본 프로젝트는 청소년 및 청년층의 우울감, 무기력증 및 사회적 고립 위험을 조기에 탐지하고 예방하는 데 초점을 맞추고 있습니다. **KLUE-BERT**와 **KoELECTRA** 모델을 한국어 웰니스 데이터에 파인튜닝하여 미세 감정 상태를 인지하고, 이를 **GPT-4o-mini** 챗봇에 연동하여 고도의 맞춤형 공감 상담을 실시간으로 제공합니다. 또한 임상 우울증 척도인 **PHQ-9 검진** 및 **상담사 전용 대시보드**를 통합적으로 구축하였습니다.

## 📊 데이터셋 구성 (Dataset Configurations)

본 연구에서는 웰니스 대화 스크립트, 일반형 챗봇 데이터셋, 그리고 일상 대화 말뭉치 데이터를 상호 결합하고 클래스 가중치를 최적화하기 위해 총 4가지 형태의 전처리 데이터셋을 설계하여 실험하였습니다.

| 데이터셋 구분 | 우울 발화 수 | 일상 발화 수 | 데이터셋 특징 |
| :--- | :---: | :---: | :--- |
| **Dataset 1** | 19,666 | 23,000 | 일상 대화 및 웰니스 스크립트를 통합한 대규모 도메인 데이터 |
| **Dataset 2** | 19,666 | 18,000 | 우울과 일상 비율을 거의 1:1에 가깝게 맞춘 균형 데이터셋 |
| **Dataset 3** | 19,666 | 1,000 | 일상 클래스를 최소화하여 우울의 세부 intent 비율과 1:1 매칭 |
| **Dataset 4** | 19,666 | 6,261 | 일상 대화 및 챗봇 말뭉치 데이터를 조율한 최적 비율 데이터셋 |

## 🛠️ 데이터 전처리 및 형태소 분석 기법 (Preprocessing)

사용자의 비정형 구어체 발화에서 감정의 왜곡을 방지하고 예측 성능을 극대화하기 위해 다단계 특화 전처리 파이프라인을 구축하였습니다.

1. **텍스트 정제**: 문장 내 감정을 암시하는 핵심 문장 부호(`?`, `!`, `.`, `,`)는 유지하고 불필요한 특수문자 및 기호 정제
2. **구어체 -> 표준어 매핑 사전 (Slang Mapping)**:
   * 단답형 축약이나 철자 왜곡(예: `주글래` $\rightarrow$ `죽고 싶어`, `자살할래` $\rightarrow$ `자살충동`)을 고위험 감지 키워드 및 감정 보정에 적합한 표준어로 변환하는 사전 구현
3. **한국어 맞춤법 교정**: 오픈소스 라이브러리인 `py-hanspell`을 연동해 비격식 발화의 오타 및 띄어쓰기 정제
4. **Kiwi 형태소 기반 맞춤형 토크나이저**:
   * **명사 중심 필터**: 감정 분류의 가시성과 분석 성능 향상을 위해 NNG(일반명사), NNP(고유명사) 추출 중심 설계
   * **감정 방향성/강도 보존**: 부정어(`안`, `못`, `없`, `아니`)와 감정강조어(`너무`, `정말`, `매우`, `엄청`)는 형태소 분석 시 제거하지 않고 예외 보존하여 해석력 극대화
   * **접두사(XPN) 자동 결합**: `무-` + `기력` $\rightarrow$ `무기력`, `불-` + `안정` $\rightarrow$ `불안정`과 같이 형태소가 강제로 분절되는 문제를 접두사와 결합하는 코드 로직으로 원형 복원
5. **한국어 욕설 감지 필터 (Profanity Filter)**: 
   * 공백 오염 및 자음 분절(예: `ㅅ ㅂ`, `씨 발`)을 모두 대응하기 위해 공백을 제거한 텍스트에서 비속어 자/모음 필터 작동

## 🤖 모델링 및 최종 모델 선정 (Modeling & Evaluation)

### 1. 실험 설계 및 후보 모델
* **KLUE-BERT-base** 및 **KoELECTRA-base-v3** 두 가지 언어 모델을 선정하였습니다.
* 가공된 4개의 데이터셋(Dataset 1 ~ 4) 각각에 대해 10 Epoch 동안 훈련을 진행하여 검증 성능을 비교하였습니다.
* 특정 감정에 과적합되는 것을 막고, 예측 확률의 신뢰도를 확보하기 위해 **검증 손실값 (Validation Loss, Val Loss)**을 최종 모델 선정의 핵심 지표로 보았습니다.

### 2. 모델별 실험 결과 (10 Epoch 기준)

#### A. KLUE-BERT 성능표
| 데이터셋 | 검증 정확도 (Val Acc) | 검증 손실 (Val Loss) | 검증 F1 (Val F1) | 테스트 F1 (Test F1) |
| :--- | :---: | :---: | :---: | :---: |
| Dataset 1 | 86.39% | 0.6305 | 86.89% | 86.31% |
| **Dataset 2 (최종 선정)** | **87.42%** | **0.6074** | **87.76%** | **86.90%** |
| Dataset 3 | 83.53% | 0.5660 | 83.53% | 83.33% |
| Dataset 4 | 86.58% | 0.5522 | 86.75% | 85.86% |

#### B. KoELECTRA 성능표
| 데이터셋 | 검증 정확도 (Val Acc) | 검증 손실 (Val Loss) | 검증 F1 (Val F1) | 테스트 F1 (Test F1) |
| :--- | :---: | :---: | :---: | :---: |
| Dataset 1 | 89.26% | 0.7323 | 89.52% | 88.68% |
| Dataset 2 | 87.63% | 0.7337 | 87.99% | 86.71% |
| Dataset 3 | 83.68% | 0.6598 | 83.65% | 82.80% |
| Dataset 4 | 83.90% | 0.6428 | 84.09% | 84.18% |

### 3. 최종 모델 선정 결과: 🏆 KLUE-BERT (Dataset 2)
* **선정 사유**: 두 모델 모두 Dataset 2에서 높은 정확도와 균형 잡힌 F1-Score를 보였으나, **검증 손실값(Validation Loss)** 비교 시 KoELECTRA(Val Loss: `0.7337`)에 비해 **KLUE-BERT(Val Loss: 0.6074)**가 훨씬 낮게 조율되어 안정적인 예측 성능과 일반화 성능을 입증하였습니다.
* 따라서 가장 안정적이고 오차율이 적은 **KLUE-BERT (Dataset 2)** 모델을 최종 챗봇의 감정 분류 및 우울 예측 엔진으로 선정하여 적용하였습니다.

## 💻 웹 애플리케이션 주요 기능 (Streamlit App Features)

### 1. 다중 페르소나 챗봇 (1단계 - 일상 대화)
* **GPT-4o-mini** API를 연동하여 페르소나가 각기 다른 5명의 가상 챗봇 구현:
  * **상담사 지우**: 따뜻하고 격려하는 전문 상담사
  * **고슴도치 또치**: 반말로 소통하는 귀여운 동네 친구
  * **AI 클로**: 객관적이고 논리적인 어시스턴트
  * **멘토 선생님**: 지혜를 나누는 인생 멘토
  * **개그맨 철수**: 긍정적이고 재치 넘치는 유머러스한 친구
* **다이내믹 프롬프트 인젝션**: 사용자의 성별, 연령대, 직업, 관심사 등 기본 프로필 정보와 직전 발화의 AI 감정 분류 분석 확률을 실시간으로 시스템 프롬프트에 주입하여 개인 맞춤형 공감 형성

### 2. 다차원 감정 분석 리포트 (2단계 - 감정 분석)
* 사용자의 발화에서 일상(일상 클래스 11)을 제외한 **19가지 우울/심리적 감정 확률의 누적 평균**을 실시간 산출
* Plotly 라이브러리를 통해 주요 감정 비율 파이 차트 및 19가지 감정 전체의 가로형 바 차트 시각화
* **오타 방지 적응형 워드클라우드**: 
  * 분석 데이터 신뢰도(Jin et al. 2023 / Chen 2021) 확보를 위해 **최소 3턴 이상 대화 및 누적 10단어 이상** 조건 충족 시에만 리포트 및 워드클라우드 활성화
  * 형태소 원형(Lemma) 변환을 거쳐 활용형을 통합하고, 형태소 분석기의 신뢰 지수(Score)가 낮게 평가된 불완전한 단어(오타)는 자동 제외한 후 시각화 진행

### 3. 우울증 자가검진 (3단계 - PHQ-9)
* 임상용 자가진단 검사인 **PHQ-9(우울증 선별도구)** 9문항 탑재
* 합산 점수에 따라 양호(0~4점), 경도(5~9점), 보통(10~14점), 중증(15~19점), 위험(20~27점)의 5단계 정밀 진단 제공

### 4. 고립 유형별 1:1 맞춤 케어 (4단계 - 맞춤 케어)
* PHQ-9 자가진단 및 AI 감정 분석 결과를 혼합하여 사용자 등급 판정
* 사용자가 사전 입력한 **관심사 카테고리(영화, 독서, 음악, 미술 등)에 따른 맞춤형 힐링 솔루션** 및 연령대별 가이드 매칭

### 5. 공공 의료 서비스 연계 (5단계 - 기관 연계)
* 전국 17개 시·도별 **광역 정신건강복지센터 공식 사이트 원클릭 연동**
* 보건복지부 국가정신건강정보포털, 복지로, 청년 지원 프로그램(청년 불씨 프로젝트) 링크 제공

### 6. 상담사 전용 대시보드 (전문가 메뉴)
* 상담 관리자가 사용자의 과거 감정 변화 트렌드 라인, 실시간 위기(자살충동, 자해 등) 탐지 로그 및 위급 개입 순위를 파악할 수 있는 통계 보드 제공

## 🚀 실행 방법 (How to Run)

### 1단계: 모델 다운로드 및 배치
1. 학습이 완료된 모델 가중치 파일들을 아래 구글 드라이브 주소에서 다운로드합니다:
   [KLUE-BERT Dataset 2 모델 가중치 드라이브](https://drive.google.com/drive/u/0/folders/1I5b3VqcPctEbg7D3TPhTGMeaORIchdAn)
2. 프로젝트 루트 폴더에 `saved_models` 폴더를 생성하고, 다운로드한 `KLUBERT_Dataset2` 폴더를 통째로 위치시킵니다.

### 2단계: 의존성 라이브러리 설치
```bash
pip install -r requirements
```

### 3단계: 환경 변수 설정 (OpenAI API Key)
챗봇 응답 모델인 GPT-4o-mini를 호출하기 위해 OpenAI API 키를 세팅합니다.

* **Windows PowerShell**:
  ```powershell
  $env:OPENAI_API_KEY = "sk-..."
  ```
* **Windows CMD**:
  ```cmd
  set OPENAI_API_KEY=sk-...
  ```
* **macOS/Linux**:
  ```bash
  export OPENAI_API_KEY="sk-..."
  ```

### 4단계: Streamlit 애플리케이션 실행
```bash
streamlit run mallang_최종.py
```
> 브라우저에서 `http://localhost:8501`이 자동으로 로드되며 서비스가 시작됩니다.
